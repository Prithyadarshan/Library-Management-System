# Project Title
Role-Based Library Management System (RBAC): Client-Side State Architecture and Multi-Page Access Governance
# Project Objective
The primary objective of this project is to architect a secure, lightweight, multi-page library ecosystem using client-side verification methodologies. By mapping strict user permissions using active session tokens, the system restricts structural system operations dynamically, separating workflows between Administrators and Standard Customers (Students). The system runs entirely on programmatic browser memory persistence, eliminating the dependency on heavy third-party external server databases.
# Core Features & Functional Description
# Multi-Page Security & Route Guarding
Initialization Guard Gates: Implements real-time structural authorization scripts at the header layer of all protected view modules. If a non-authorized client manually enters the localized file paths, the handler captures the exception, blocks component rendering, triggers a security warning layout alert, and forces an absolute logout redirection.
# Dynamic UI Layout Masking
Dynamic DOM Object Masking: Evaluates the client's clearance parameters upon view initialization to selectively hide code components. Administrative tools, data modification actions, and resource insertion portals are cleanly stripped from standard customer display grids.
# Client Registration & Credentials Filtering
Regex Structural Filtering: Features a dedicated account-building matrix running string validation filters, ensuring 10-digit limit checks on phone numbers, matching password confirmation validations, and duplicate identity evaluation over cached email data records.
# Comprehensive Database CRUD Engineering
Asset Indexing (Create): Equips authorized roles with clean, form-based input dashboards to insert new records into the master catalogue collection.

Persistent Display (Read): Dynamically updates and maps data lists to a structured, mobile-responsive data grid on initialization.

Data Modification (Update & Delete): Captures targeted index vectors by mapping active properties through URL search parameters to safely overwrite existing asset attributes or delete outdated entities.
# Technical Architecture Specifications
Frontend User Interface Stack: Standard semantic HTML5 structure styled with modern CSS3 elements (Glassmorphic layout blends using backdrop-filter, responsive Flexbox containers, grid formatting cards, custom background linear gradients, and clean entry keyframe animations).

Storage & Application State Engine: Pure JavaScript (ES6+ Runtime Actions) mapping structural array objects into serialized JSON string blocks using the native Web LocalStorage API.

Deployment Optimization Pipeline: Fully compiled and hosted via Vercel Cloud Infrastructure Engine, utilizing optimized file formatting flags ("cleanUrls": true) to clean static .html resource suffixes from live browser navigation bars.
# Future Roadmap Enhancements
Relational Database Migration: Move the client-side state machine arrays into a robust cloud database infrastructure (like MongoDB or PostgreSQL) via a server layer (like Node.js or Python Flask).

Search Optimization Filtering: Implement real-time string searching algorithms to allow users to filter the master inventory dataset by book title, author, or genre instantly as they type.

Advanced Security Hashing: Integrate basic cryptographic algorithms to hash user passwords locally before committing them to storage, ensuring high-grade data privacy.
